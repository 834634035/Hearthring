防抖 & 节流 演示

对比「触发次数」和「实际执行次数」，感受两种策略的差异。

防抖 Debounce — 停下来才执行

1. 搜索输入框

连续输入时不会立刻「发请求」，停手 500ms 后才执行。

触发次数: 0 实际执行: 0

2. 按钮连点

快速连点，只有停手 800ms 后才会执行一次。

触发次数: 0 实际执行: 0

节流 Throttle — 固定间隔执行

3. 滚动区域

快速滚动时，每 500ms 最多更新一次位置（第一次立即响应）。
